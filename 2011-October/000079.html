New issue 28: autosave downloaded data
https://bitbucket.org/steko/totalopenstation/issue/28/autosave-downloaded-data

Stefano Costa / steko on Sun, 9 Oct 2011 15:08:17 +0200:

Description:
  With the current user interface, it's too easy to lose data just by closing the program window, or importing other data either by downloading or opening an existing file.

We should add an autosave feature.

Downloaded data should be saved **in real time** on disk to a file (in the //.totalopenstation// folder. Real-time prevents data loss when the program crashes or if the user is forced to stop a transfer before its end.

Files could be also gzipped to save space. Filenames should just be a timestamp.tops.gz like //201110091504.tops.gz//.
