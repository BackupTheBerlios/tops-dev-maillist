No, I think returning as much explicit as possible data is better - it's
up to the "output generator" for each format to decode the raw
measurements and give the user a reasonable choice. This is what we're
already doing and it helps in keeping clearly separated input processing
and output generation.

> Btw: How to you get Python to import classes/files with a notation
> like "totalopenstation.formats.XXX.YYY"? 

totalopenstation is a top-level Python package, it has a setup.py and
each sub-package has a __init__.py file telling the Python interpreter
that that folder is a package. It's probably not needed for QGIS but
it's certainly useful in general. Feel free to copy the directory
structure from totalopenstation :-)
