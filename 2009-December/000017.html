Hi,
during the past weekend I spent some time rewriting a large deal of our
documentation pages. While it's still a work in progress (with some
pages still outdated, that is), I think it would be better to upload
them online instead of the current ones.

Using Sphinx, I found it more convenient to redesign the whole website
(with some easy color tweaking from the default theme) instead of
keeping it separate.

You can see the new documentation online at http://tops.berlios.de/new/

As said above, it still is a work in progress, and there are some points
where I can't decide myself about the level of detail we should put in
the docs, for example in the "For developers" section of documentation.
Please give your opinion about that. If there are no drawbacks, I'd like
to make those pages the default website next week.

Nevertheless, the updated documentation reflects the new infrastructure
that is in tip code (see my previous email), with its strict separation
of models and input formats. I'm also working on integrating the
documentation inside a standard Python package along with TOPS source
code (more on that later).
