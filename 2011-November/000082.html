Hi Anna,
first of all big, big apologies for not getting back to your request
last month (or was it even September?).

Just yesterday I translated to English a step-by-step guide that covers
the installation of TOPS on Windows. You can find it at
https://bitbucket.org/steko/totalopenstation/wiki/InstallingOnWindows
and I have tested it on 3 Windows XP machines (including a tablet PC).

Other installation procedures might not be correct. Especially, pySerial
2.5 (that is, the latest version that you get by default when
downloading) has a bug on Windows and it doesn't work properly. This has
been the cause of many headaches until I realised that 2.4 works like a
charm.

Before going into detailed debugging of the problem you're facing, I
would like to ask you to start over with that procedure (possibly
including installing a newer Python interpreter ‒ you will find links
for that too in the guide).

> I would be grateful if someone could send me some sample data that can
> be imported into TOPS, by the way, preferably some survey daya with
> lines and / or polygons - I would like to export a dxf from TOPS and
> see if the line data comes through. 
> Stefano, you mentioned a possibility of exporting data from TOPS
> without having to use the DXF2SHP converter in Qgis - I would be
> interested to know what is done instead.

You can get some sample data here
https://bitbucket.org/steko/totalopenstation/src/9609c1e38c14/sample_data ‒ I have some more data from Nikon and Leica devices.

The total station I'm working with has no support for lines and/or
polygons, just bare points, so I've never looked into creating them from
the raw data. But it's definitely possible to enhance the TOPS parser to
take it into account.

Exporting data from TOPS straight into QGIS is possible if you install
the "geojson" Python package http://pypi.python.org/pypi/geojson/ (no
Windows installer though, I'm afraid). Then, a new output format becomes
available and it's OGR-compatible: QGIS can open GeoJSON data saved into
a plain text file.
